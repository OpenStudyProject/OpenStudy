技術ブログやQiita、Zennなどの場合

基本的に1つの記事は1人が書いたもの

  • 情報が断片的になる
  • 他の人の記事を修正できない
  • 情報が重複するため書き手の労力が増える
  • 読み手が目的の情報を見つけるのに時間がかかる

OpenStudyでは

{{ .Site.Data.openstudy_logo.svg | safeHTML }}

すべての人が既存の記事に追記、修正もできる

強力な知識のデータベースです

少数の人がすべてを網羅するよりも、多数の人が各々の得意分野を書いた方が圧倒的に有利であり、効率もいいはずです。記事が編集されればされるほど、多くのエンジニアの知識が凝集した、正確かつ有用で価値のあるページになります。

パワフルな Hugo とシンプルな Markdown

高速な静的サイトジェネレーターのHugoを使用しているため、記事の分類やタグ付けが非常に簡単です。多くの人の共同執筆でありながらシンプルなため、記事を書くことだけに集中できます。

# 見出し

```text
# 大見出し
## 見出し
### 小見出し
```

OpenStudyでは、HTMLにレンダリングされるとそれぞれが に対応します。目次に表示されるのは大見出しと見出しのみです。

# リスト

## 番号なしリスト

ハイフン`-`、プラス`+`、アスタリスク`*`のいずれかを先頭に記述します。スペース4つでネストできます。
